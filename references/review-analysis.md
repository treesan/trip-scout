# 差评分析引擎

## 核心原则

**不是所有差评都一样重要。** 成长性差评可以忽略，本质性差评必须拉黑。

## 差评分类矩阵

| 类型 | 关键词 | 处理方式 | 信任影响 | 示例 |
|------|--------|----------|----------|------|
| 🟢 **成长性差评** | 装修味、新开业、服务响应慢、磨合期、还在调试 | **忽略** — 随时间改善 | 无 | "刚开业有点味道"、"服务还在磨合" |
| 🟡 **可接受差评** | 排队久、价格贵、停车远、位置偏、隔音一般 | **记录** — 不影响推荐 | 无 | "节假日排队1小时"、"停车费贵" |
| 🔴 **本质性差评** | 卫生问题、态度恶劣、安全隐忧、设施严重不符、欺诈、强制消费 | **拉黑** + 通知用户 | 降品牌信任 | "房间有蟑螂"、"前台态度极差"、"实际与图片严重不符" |
| ⚫ **系统性差评** | 加盟管理混乱、频繁换名、挂牌卖质、管理方频繁更换 | **拉黑** + 标记加盟风险 | 降品牌信任 | "原来是XX酒店改名的"、"管理很乱" |

## 关键词库

### 🟢 成长性关键词（可忽略）

```
装修味 / 甲醛 / 新开业 / 还在磨合 / 服务还在提升 / 还在调试
设施还在完善 / 员工培训中 / 刚开业不久 / 开业才X个月
```

### 🟡 可接受关键词（记录不拉黑）

```
排队 / 等待久 / 价格贵 / 性价比低 / 停车远 / 停车不方便
位置偏 / 交通不便 / 隔音一般 / 房间小 / 早餐一般
 wifi慢 / 空调声音大 / 水压低
```

### 🔴 本质性关键词（拉黑）

```
脏 / 卫生差 / 有虫 / 蟑螂 / 老鼠 / 异味(非装修)
态度差 / 态度恶劣 / 服务差 / 不处理 / 推诿
不安全 / 门锁坏 / 消防 / 安全隐患
图片与实际严重不符 / 欺诈 / 强制消费 / 临时涨价
设施损坏 / 热水没有 / 空调坏了 / 电视坏了
```

### ⚫ 系统性关键词（拉黑+加盟标记）

```
原来是XX酒店 / 原名XX / 改名 / 翻牌 / 换了管理方
管理混乱 / 管理方换了 / 员工流动大 / 经常换人
挂大牌卖小店 / 品牌标准不符 / 和官网图片差太多
加盟 / 个体经营
```

## 分析流程

```
收到酒店评价
  │
  ├─ 1. 提取差评内容
  │
  ├─ 2. 关键词匹配
  │   ├─ 匹配🔴本质性 → 触发拉黑流程
  │   ├─ 匹配⚫系统性 → 触发加盟识别
  │   ├─ 匹配🟢成长性 → 标记"可忽略"
  │   └─ 匹配🟡可接受 → 标记"记录"
  │
  ├─ 3. 统计各类占比
  │   └─ 成长性差评占比 > 50% → 整体评价上调
  │
  ├─ 4. 输出分析结果
  │   └─ 写入搜索结果的"差评分析"字段
  │
  └─ 5. 触发后续动作
      ├─ 本质性差评 → 写入 blacklist.md + 降品牌信任
      └─ 系统性差评 → 标记加盟风险 + 检查改名记录
```

## 携程差评量化规则(强制, 防以偏概全)

不能只凭问道摘要(代表性1-2条)下结论, 必须按以下量化标准抓取与阅读:

### 时间范围与条数

| 维度 | 规则 | 理由 |
|------|------|------|
| **时间范围** | 近 **12 个月** | 近期口碑最能预测入住体验; 一年前可能已改善/恶化, 权重低 |
| **差评条数(评价量>500)** | 读差评前 **30-50 条** | 大店差评多, 取近一年最相关的前30-50条分类 |
| **差评条数(评价量<500)** | 读 **全部差评** | 小店差评少, 全读避免漏 |
| **近期趋势** | 单看近 **3 个月**差评突增 | 突增=近期恶化⚠️(如新换管理/翻牌), 即使近12个月整体尚可也要警示 |

### 获取路径(按优先级)

1. **脚本抓取**(主, 推荐): `python scripts/ctrip_reviews.py <hotelId> --months 12 --pages 20`
   - 复用携程内部 REST API `getHotelCommentList`(源自 [Z-an-K/ctrip-review-crawler](https://github.com/Z-an-K/ctrip-review-crawler), MIT), 纯 requests 无浏览器。
   - `filterInfo id:3` 精准抓差评, 按时间排序, 含 rating/createDate/content。
   - 脚本自动分类(成长性/可接受/本质性/系统性)+统计占比+近3月趋势+踩雷风险。
   - hotelId 从携程酒店详情页 URL 取(`hotelid=XXXX`)。
2. **携程问道API**(补充): 问"近一年差评分类、典型问题"拿摘要, 与脚本结果交叉。
   - ⚠️ 问道只给代表性摘要, 可能漏差评; 会话常被锁定。
3. **opencli-browser**(兜底): 脚本失败/被风控时, 登 ctrip.com 手动翻页。

### 两边都负面时的决策

原酒店A 和 备选B 都有负面 → **不是选负面少的, 而是选本质性差评轻的**:

```
比较 A 和 B 的:
  1. 差评类型: 成长性(可忽略) < 可接受(记录) < 本质性(拉黑) < 系统性(拉黑+加盟)
  2. 本质性+系统性差评占比(条数/总评价量)
  3. 严重度: 卫生/安全 > 服务 > 设施
  4. 时效性: 近3个月 > 近12个月 > 更早(近期权重高)
→ 选占比低、严重度轻、非近期恶化的一家
→ 若都严重 → 两家都放弃, 扩大搜索找第三家
```

## 特殊场景处理

### 新开业酒店

- 开业 < 6个月: 成长性差评关键词自动匹配
- 开业 < 12个月: 装修味/服务磨合类差评降权
- 开业 > 12个月: 正常分析

### 加盟翻牌店

- 发现改名记录 → 自动标记为加盟翻牌
- 检查改名前的差评 → 作为当前酒店的参考
- 用户已拉黑的翻牌店 → 同品牌同城同级别全部跳过

### 评分异常

- 同品牌不同店评分差异 > 1.0 → 标注"品控不稳"
- 评价量 < 100 → 标注"评价不足, 参考价值有限"
- 近30天差评突增 → 标注"近期口碑下降"
