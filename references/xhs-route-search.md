# 小红书路线搜索工作流（自驾游路线推荐专用）

> 与 `xhs-hotel-research.md`（酒店口碑验证）共享 `scripts/xhs.py` 搜索能力，但搜索词策略、筛选标准和信息提取完全不同。

## 目的

从小红书搜索经典自驾游路线攻略，提取路线骨架（每日行程、住宿点、驾驶里程），结合用户画像（双家庭、亲子友好、驾驶节奏）筛选出适合的候选路线。

## 搜索词模板（路线搜索专用）

与酒店口碑搜索（精准词"酒店名+避雷"）不同，路线搜索使用**泛词**：

```
"{区域} 自驾游路线"               # 🥇 首选，最泛最广
"{区域} {天数}天 自驾游"           # 按天数过滤（如"川西 3天 自驾游"）
"{区域} 亲子自驾游"                # 亲子过滤
"{具体路线名} 路线图"              # 如已知路线名（如"川西小环线 路线图"）
"{区域} 自驾游 避坑"              # 避坑版，看踩雷经验
"{出发城市} {目的地} 自驾游"       # 起终点组合（如"成都 新疆 自驾游"）
"{区域} 自驾游行程表"              # 搜索有行程表的攻略
"{区域} 自驾游地图"                # 搜索有地图的攻略
```

搜索策略：
1. 先跑首选模板 "{区域} 自驾游路线"，获取广覆盖
2. 如结果不够，追加天数过滤/亲子过滤模板
3. 如用户提到具体路线名，优先搜该路线名
4. 避坑版用于验证候选路线的踩雷风险

## 调用方式

```bash
# 确认登录（失效则 python scripts/xhs.py --show qrcode）
python scripts/xhs.py check-login

# 搜路线攻略（--show 必须，无头模式导航超时）
python scripts/xhs.py --show search "川西小环线 自驾游路线" --limit 20
python scripts/xhs.py --show search "川西 3天 自驾游" --limit 15
python scripts/xhs.py --show search "川西 亲子自驾游" --limit 10

# 取最相关的 feed 详情
python scripts/xhs.py --show feed <feed_id> <xsec_token>
```

## 筛选标准（与酒店口碑验证完全不同）

酒店口碑验证：过滤软文找真实体验，搜的是单店踩雷信号
路线搜索：找路线图清晰+行程合理+有参考价值，搜的是行程骨架

### 保留（路线信号）

- **有完整路线图/行程表**：每天去哪、住哪、开多少公里
- **住宿点明确**：能提取出每晚住宿的城市/区域
- **有驾驶里程/时长**：写了每天开多久、多远
- **有亲子相关评价**：提到儿童体验、适合不适合带孩子
- **有路况提示**：山路、区间测速、冬季封路等实用信息
- **有费用估算**：油费、过路费、住宿费等

### 丢弃（非路线信号）

- **纯种草无行程细节**：只写"太美了一定要去"，没有行程骨架
- **只有风景照无行程**：图片好看但文字没有路线信息
- **广告/旅行社推广**：旅行社推销固定套餐，非个人真实体验
- **过于小众非主流路线**：极端探险路线，不适合双家庭亲子
- **搬运/抄袭**：明显搬运其他博主内容
- **商业软文**：酒店/景区推广帖，伪装成攻略

### 判定规则

```
单条笔记:
  路线信号 ≥3 条 → 采纳（有行程表+住宿点+里程=3条，已够）
  路线信号 ≥2 条 + 非路线信号 ≤1 → 采纳（有行程骨架但不够详细，仍可参考）
  非路线信号 ≥2 条 → 丢弃（纯种草/广告）
  两者都不足 → 边际，仅作灵感参考

整体路线候选:
  至少3条笔记指向同一条路线 → 路线成立（多人验证=可信）
  1-2条笔记提到 → 标注"小众路线，验证少"，降低优先级
```

## 信息提取格式

从采纳的路线笔记中提取路线骨架，写入行程规划底料：

```markdown
## 📌 路线搜索结果：{路线名}

### 来源笔记
- 笔记1: {标题} | {链接} | 路线信号: {行程表+住宿点+里程}
- 笔记2: {标题} | {链接} | 路线信号: {路况提示+亲子评价}
- 丢弃笔记: X条（原因：{纯种草/广告/搬运}）

### 路线骨架
| 天数 | 行程 | 住宿点 | 驾驶里程 | 驾驶时长 | 路况 | 亲子友好度 |
|------|------|--------|---------|---------|------|-----------|
| D1   | {出发→目的地} | {城市} | {km} | {h} | {高速/国道/省道} | {评价} |
| D2   | ... | ... | ... | ... | ... | ... |

### 关键发现
- 路况: {山路/测速/封路提示}
- 亲子: {儿童体验、适合不适合}
- 费用: {油费/过路费估算}
- 避坑: {踩雷经验}

### 验证状态
- 笔记条数: X条采纳（Y条丢弃）
- 路线验证: ✅多人验证 / ⚠️验证少 / ❌不可行
```

## 与酒店口碑搜索的关键差异

| 维度 | 酒店口碑搜索（xhs-hotel-research.md） | 路线搜索（本文档） |
|------|--------------------------------------|------------------|
| 搜索词 | 精准："酒店名 避雷" | 泛词："区域 自驾游路线" |
| 结果类型 | 单店口碑（踩雷信号） | 行程攻略（路线骨架） |
| 筛选标准 | 过滤软文找真实体验 | 找路线图清晰+行程合理 |
| 保留信号 | 具体问题、重复槽点 | 行程表、住宿点、里程 |
| 丢弃信号 | 通篇赞美无缺点 | 纯种草无行程 |
| 提取信息 | 踩雷信号+游玩反哺 | 每日行程骨架+路况+亲子 |
| 排序 | 负面信号优先（避雷） | 完整度+亲子友好+参考价值 |
| 收敛 | 热门≥5/小众≥3，信号一致 | 至少3条指向同路线 |

## 路线搜索后的下一步

路线搜索提取的骨架 → 交给"场景二 Phase 1: 路线推荐"做：
1. 结合用户画像适配（双家庭→亲子友好筛选、每天≤300km→砍超标天数）
2. 高德API校正里程/时长/过路费
3. 快速酒店可行性检查（每个住宿点有无合格酒店）
4. 输出2-3条候选路线供用户选择

## 常见坑

1. **泛词搜到大量种草** — "川西自驾游路线"搜到的前10条可能全是风景照无行程，需要耐心翻和筛选
2. **行程表格式不统一** — 小红书笔记的行程表格式各异（文字/表格/图片），需要灵活提取
3. **里程数据不精确** — 笔记中的"约300km"是大致估算，必须用高德API校正
4. **亲子信息缺失** — 很多自驾游攻略不提亲子体验，需要额外搜"亲子"模板补充
5. **路线过时** — 部分路线笔记是1-2年前的，路况可能已变化，需搜最新版验证
