# 航班搜索工作流

## 触发条件

用户提到机票、航班、出发/到达相关关键词：
- "搜一下 7月15号 上海到杭州的机票"
- "查一下明天成都飞北京的航班"
- "flight search Beijing to Shanghai"

## 输入提取

从用户输入中提取：

| 字段 | 说明 | 必填 |
|------|------|------|
| origin | 出发城市/机场 | ✅ |
| destination | 目的地城市/机场 | ✅ |
| date | 出发日期 | ✅ |
| back_date | 返程日期 | ❌ |
| passengers | 人数 | ❌ (默认2大人) |
| direct_only | 是否只看直飞 | ❌ |

## 双平台并行查询

### 飞猪 (flyai)

```bash
flyai search-flight \
  --origin "上海" \
  --destination "杭州" \
  --dep-date 2026-07-15 \
  --sort-type 3
```

关键参数：
- `--sort-type 3` = 价格升序
- `--journey-type 1` = 直飞
- `--max-price` = 价格上限

返回格式：JSON，包含航班号、时间、价格、航司等

### 携程 (wendao)

```bash
jq -n \
  --arg token "$WENDAO_API_KEY" \
  --arg query "搜索7月15号上海到杭州的航班" \
  '{token: $token, query: $query}' | \
  curl -s -X POST https://wendao-skill-prod.ctrip.com/skill/query \
  -H "Content-Type: application/json" \
  -d @- > /tmp/wendao-flight.json
```

查询模板：
- "搜索{日期}{出发}到{目的地}的航班"
- "搜索{日期}{出发}到{目的地}的直飞航班"
- "搜索{日期}{出发}到{目的地}的航班，经济舱"

返回格式：Markdown 正文，需提取航班信息

## 结果合并与排序

1. 从两个平台提取航班信息
2. 按航班号匹配（同一航班对比价格）
3. 按价格升序排列
4. 标注：直飞/转飞、航司、是否有餐、行李额

## 输出格式

```markdown
## ✈️ [出发] → [目的地] | [日期]

| 平台 | 航班 | 时间 | 价格 | 备注 |
|------|------|------|------|------|
| 飞猪 | MU5101 | 08:00-10:30 | ¥680 | 直飞, 东航 |
| 携程 | MU5101 | 08:00-10:30 | ¥720 | 直飞, 可选座 |

### 💡 建议
飞猪价格更低(省¥40)。建议选飞猪预订。
```

如果两个平台价格差异 < ¥20：
```markdown
### 💡 建议
价格接近，携程可选座位。建议选携程。
```

## 错误处理

- 飞猪无结果 → 只展示携程结果，标注"飞猪未找到"
- 携程 API 超时 → 重试1次，仍失败则只展示飞猪结果
- 两个都失败 → 提示用户稍后重试或手动查询
