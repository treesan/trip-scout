`~/.trip-scout/MEMORY.md`

首次运行时自动创建，包含用户偏好和入住历史。

## 完整格式

```markdown
# Trip Scout Memory

## 出发城市
- 默认: 成都

## 酒店偏好
- 品牌梯度: 万豪系🟢 > IHG🟡 > 万达系🟠 > 亚朵 > 雅高 > 橘子水晶 > 全季
- 温德姆规则: 4星级仅允许直营戴斯, 加盟一律跳过
- 价格区间: 经济型¥500-700 | 品质¥700-1200 | 轻奢¥1200-1800
- 开业时间: 2020年后优先
- 携程评分: ≥ 4.7
- 必需设施: 亲子设施(泳池/儿童乐园)

## 航班偏好
- 舱位: 经济舱
- 直飞优先: 是
- 时段: 无偏好

## 会员权益
- 携程: 钻石会员
- 万豪: 金卡(飞猪平台可用, **房价不打折不送早**, 仅升房+餐饮85折+延迟退房)
- 飞猪: F3/F4直通万豪金卡、希尔顿金卡等
- 亚朵: 金卡(APP有早餐券)
- 华住会: 企业铂金会员(85折+送双早)

## 会员价取值规则(比价核心, 必须遵守)
- 亚朵金卡: 有早餐券 → 取**携程不含双早价**作为金卡实付价(达到含早效果)
- 华住会企业铂金: 房价85折+送双早 → **携程不含双早价 × 0.85** 作为铂金实付价(含早)
- 万豪金卡: 房价不打折不送早 → 直接用**携程/飞猪含双早价**(不折算, 价值在升房+餐饮85折)
- 通则: 统一折算到**含早实付价**再比较。万豪用含早挂牌价, 亚朵/华住用不含早价折算。

## 比价房型基准(强制)
- 默认房型按出行人员推断: 两大一小=双床房, 两大两小=家庭房, 情侣=大床房(用户可覆盖)
- 比价必须统一取**双床房(或同房型)+含双早**实付价, 不可比不同房型/不同含早状态

## 学到的规则
1. 温德姆 4 星级加盟店风险高, 仅允许直营戴斯
2. 新开业酒店装修味差评可忽略, 随时间改善
3. 万豪系直营品质最稳定, 同价位优先推荐
4. 万达系中端(锦华/美华)在偏远城市性价比极高
5. 加盟翻牌店需检查历史改名记录
6. 首次触发亚朵酒店 → 先确认用户是否有亚朵金卡(金卡价≈携程不含双早价)
7. 首次触发华住系酒店(桔子水晶/全季) → 先确认华住会会员等级(企业铂金=85折+送双早)
8. 价格相差不大(±5%)时, 按信任梯度+口碑+位置+环境+服务综合判断, 不唯价格
9. 赛里木湖景区内住宿2026已全关, 只选东门正规酒店
10. 问道API受限时降级: opencli-browser登录ctrip.com抓房型价格评价
11. 比价房型强制统一: 双床房(按出行人员推断)+含双早实付价, 不同房型/含早状态不可比
12. 万豪金卡房价不打折不送早, 比价直接用含早挂牌价(价值在升房+餐饮85折, 不省房费)
13. 目的地有五星必须先纳入五星基准跑完整流程再定四星(曾D4伊宁失误漏绿发洲际)
```

## 自进化规则

### 信任度计算

```python
def update_trust(brand, experience, is_franchise):
    if experience == "bad" and is_franchise:
        # 加盟翻牌差体验 → 降品牌信任
        brand.trust_level -= 2
        brand.franchise_risk += 1
        add_to_blacklist(brand, "加盟翻牌差体验")
    
    elif experience == "bad":
        # 直营差体验 → 轻微降级
        brand.trust_level -= 1
    
    elif experience == "good" and brand.trust_level < "高":
        # 好体验 → 逐步升级
        brand.trust_level += 1
    
    # 检查是否需要降级整个品牌线
    if brand.franchise_risk >= 2:
        brand.trust_level = min(brand.trust_level, "低")
```

### 规则学习

每次用户反馈后，检查是否能提炼新规则：

```
用户反馈: "XX酒店的XX品牌加盟店不行"
  │
  ├─ 检查是否已有类似规则 → 有则强化, 无则新增
  │
  └─ 新规则示例:
      "XX品牌加盟店风险高, 后续搜索时需验证直营"
```

### 规则优先级

1. 用户明确规则 > 系统默认规则
2. 近期规则 > 历史规则
3. 多次验证的规则 > 单次经验
